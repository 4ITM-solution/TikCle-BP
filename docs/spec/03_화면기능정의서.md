## 0. UX 원칙 (2026-07-07 사용자 확정 — 모든 화면 작업의 상위 규칙)

| # | 원칙 | 구체 규칙 |
|---|---|---|
| U1 | **프리셋 우선, 토글은 숨김** | 토글 곱연산(채널×월×측정치×기간×SKU) 금지. 섹션마다 가장 많이 보는 조합 1개를 디폴트 렌더, 나머지는 "자세히" 뒤로 |
| U2 | **답 먼저, 차트는 근거** | 모든 블록 상단 1줄 결론 필수 (예: "광고의 70%가 Amazon행 = Amazon 퍼널 중심"). 결론 못 만드는 블록은 회색 "데이터 없음" |
| U3 | **사용자 언어만** | phase 코드·"옛 MD"·도구명(Exolyt 등)·skipped_reason 코드 화면 노출 금지. 용어 사전: phase → "영상 태깅 중" 등 |
| U4 | **TOC = 질문 문장** | A/B/C/D/E 알파벳 대신 "언제 터뜨렸나 / 누구를 썼나 / 뭘 찍었나 / 팔렸나 / 광고는 어떻게" |
| U5 | **숫자의 신뢰도 표기** | 추정치는 `~` 접두(예: ~$12K), 캐시/라이브 구분 배지, 갱신 시점 표기. G1~G8 갭(재설계 §2.1)의 라벨 규칙 준수 |
| U6 | **paid ≠ 유가** | "paid" 라벨 전면 "광고 집행(스파크애즈)"로 교체. 유가 규모 논의는 티어 분포로 (마이크로+ ≈ 유가 — 사용자 확정) |

## 0.1 적재 위저드 [WS4 신규 — "데이터 적재가 오래 걸리고 헷갈린다" 해소]

현재: 채널 카드 9개 병렬 나열, 순서·출처·진행률 안내 없음. 목표:

```
케이스 생성(브랜드+국가) 직후
├─ 자동 수집 즉시 시작 배너: "메타광고·IG·YT 수집 중" (사람을 기다리지 않음 — §1.0.3)
└─ 수동 재료 체크리스트 (케이스의 country×channel로 자동 생성):
     ① Exolyt CSV — 출처 안내 링크 + 예상 소요 (~10분)   ✅
     ② Helium 매출 CSV (~5분)                          ⬜
     ③ Keepa BSR (~5분)                                ⬜
     ④ Kalodata 3종 (TT샵 케이스만, 365일 기준)          ⬜
     ─────────────────────────────
     진행 n/N · "지금 올린 것으로 답할 수 있는 것: 완결성 6축 중 ②③" ← 게이지 연동
```

- 각 항목: 출처 사이트·export 방법 1줄·업로드 직후 "n행 적재됨" 즉시 피드백.
- 업로드가 늦어도 자동 수집분으로 섹션이 먼저 열림 (ready 게이트 완화와 한 세트).
