# 04 아웃풋 명세 — 브리프 3종 + 진단-매칭 운영안

> 제품 정의: BP_재설계_v2 §1.4. 아웃풋은 3종 — ①대시보드(03 문서) ②GTM 콘텐츠(브리프) ③진단-매칭 운영안. 이 문서는 ②③의 입출력 계약.

## 공통 원칙

1. **수치는 LLM이 생성하지 않는다.** 모든 숫자는 뷰/테이블에서 SQL로 뽑아 템플릿에 주입하고, LLM은 서술·연결만 담당. 브리프 하단에 근거 쿼리 시점 명시.
2. 완결성 6축(§1.0.2) 미달 축의 내용은 브리프에서 생략하거나 "데이터 없음"으로 명시 — 추정으로 채우지 않는다.
3. 산출물은 markdown 우선 (슬랙 복붙 호환). html/슬라이드는 후순위.

## 1. 브리프 3종 (가공 A — WS7)

### 입력 (3종 공통 — "브리프 컨텍스트")

케이스 1개에 대해 SQL로 조립하는 JSON:

| 필드 | 소스 |
|---|---|
| 규모: 기간·영상수·크리에이터수·채널 구성 | v_case_monthly, v_case_creator_stats |
| 티어 믹스 (채널별) | v_case_tier_dist |
| 반복 협업 top (2회+) | v_case_creator_stats |
| 앵글 top 5 + 대표 영상 | content_clusters(+members) |
| 성과: BSR 추이 요약·변곡점·GMV/매출 | sales_snapshot, case_insights(bsr_inflections), kalodata |
| 광고: 3분류 비중·최장 운영 소재·origin | meta_ads, v_case_ad_runtime(WS3) |
| 시딩∩광고 인플 | v_case_seeding_ad_overlap(WS4) |
| core_factor (사람 확정값) | cases.core_factor(WS6) — 미확정이면 브리프에 "미확정" 표기 |

### 출력 3종

| 종류 | 용도 | 형태 | 톤 |
|---|---|---|---|
| **슬랙 요약** | "지역별 BP 요약 남겨줘" 류 즉답 | 5±2줄 불릿, 팩트 위주 ("무가시딩 위주, 물량 상한 있음" 급 압축) | 내부 슬랙체 |
| **세일즈 Appendix** | 피칭 레퍼런스 ("이렇게 해서 성공한 게 X입니다") | 1~2p: 배경→무엇을 했나(6축 순)→성과→core_factor | 대외 존댓말 |
| **미팅 브리프** | 특정 미팅 대비 ("닥터포헤어 미팅 준비") | 예상 질문 → 케이스 근거 답변 매핑 | 내부 |

톤·구조 레퍼런스: 월간 사보 html, 닥터포헤어 미팅 html, 4am.team/blog — **WS7 착수 시 수집·분석 후 템플릿 확정** (이 문서에 반영).

### UI 접점
케이스 상세 헤더에 "브리프" 버튼 → 3종 선택 → 생성 → 복사. 생성 이력은 저장하지 않음(v1).

## 2. 진단-매칭 운영안 (가공 B — WS8)

### 입력
tikcle-result 진단 응답: `{국가, 판매채널, 월예산, 스테이지(진입전/초기/성장), 카테고리}` (기존 진단 도구가 이미 구조화 수집 중).

### 처리 (3단계)

```
1) 필터: country × channel × 예산티어 → 완결성 게이지 상위 케이스 후보 (WS6 필터)
2) 판정: 후보 케이스의 core_factor(사람 확정 필수)와 응답 예산/스테이지 대조
   - 충족 → "유사 케이스 + 이 믹스·타임라인·앵글" 운영안
   - 미달 → "이 방식은 불가 — 코어가 [X]라서" + (가능하면) 축소 실행안
   - 매칭 케이스 없음 → "해당 조합 케이스 미보유" (지어내지 않음, 커버리지 갭으로 기록)
3) 생성: 운영안 문서 (믹스 = 티어×인원수, 타임라인 = 진입 N개월차 축, 소재 = 앵글 시퀀스)
```

### 출력
`{매칭 케이스 요지(익명화 옵션), 판정(가능/축소/불가+사유), 운영안(믹스·타임라인·소재), 근거 수치}`.

### 제약
- core_factor가 사람 확정되지 않은 케이스는 매칭 후보에서 제외 (검증 안 된 판단 금지).
- 선행: WS6(필터·core_factor) + WS7(생성 엔진 공용화). 진단 도구와의 연동 방식(수동 트리거 vs 자동)은 WS8 착수 시 결정.
