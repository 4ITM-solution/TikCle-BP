## WS7 — GTM 발행 레이어 (가공 A)

**공통 파이프 (불변)**: ①소스(BP 뷰 실측 — 수치는 전부 SQL, LLM은 서술만) → ②기획 md [사람 확정] → ③코드 생성 → ④폴리시 [사람 확정]

### 7-1. 표준 쿼리 세트 `src/lib/gtm/queries.ts`
발행물이 공통으로 쓰는 read-only 쿼리 함수 모음 (전부 기존 뷰 위): 케이스 요약(6축·freshness·core_factor), 지역/언어권 컷(국가군별 콘텐츠 수·티어 풀·나노 vs 메가 성과·케이스 목록), 클러스터 top·BSR 변곡 한문장·광고 랭킹. **여기가 "클로드가 Supabase 직결로 쓰던 즉석 쿼리"의 표준화 지점** — 스페인어권 덱 때 반복 작성한 쿼리들을 함수로 박제.

### 7-2. 케이스 브리프 생성기 (대시보드 "브리프 복사" 버튼)
- 3종: 슬랙 요약(5줄) / 세일즈 Appendix(케이스 스토리) / 미팅 브리프(질문 대응형).
- 서버 액션 1콜(Sonnet): 입력 = 7-1 쿼리 결과 JSON(수치) + 템플릿. **모든 수치는 템플릿 주입, LLM 생성 수치는 렌더 전 정규식 검출 시 실패 처리.**
- freshness 배지(WS6 §5) 필수 포함 — "이 브리프의 광고 데이터는 N일 전 기준".
- 톤 레퍼런스: docs/gtm-assets/tikcle-letter/SKILL.md 톤 규칙 + 4am.team/blog.

### 7-3. 기획 md 템플릿 2종 (`docs/gtm-assets/templates/`)
- `플레이북_인덱스_TEMPLATE.md` / `플레이북_덱기획_TEMPLATE.md` — 스페인어권 실물의 구조를 변수화 (섹션↔데이터 매핑 표 + "부족정보" 체크리스트 필수 포함).
- 레터용: tikcle-letter 스킬의 딥다이브 코너 입력 스키마에 맞춘 `레터_딥다이브_TEMPLATE.md` (브리프 생성기 출력이 바로 들어가게).

### 완료 기준
실케이스 1개(SharkNinja 또는 닥터리쥬올)로: 브리프 3종 생성(수치 라이브 검증) + 레터 딥다이브 1편(템플릿 경유) + 지역 컷 쿼리로 덱 기획 md 1개 초안. 사람 체크포인트 2곳이 어디인지 명시.

## WS8 — 진단-매칭 운영안 (가공 B)

- 입력: tikcle-result 진단 응답(국가·판매채널·예산·스테이지 — 이미 구조화됨).
- 처리: WS6 `case_adoption` 필터(Q0) → `v_case_ranking`(Q7) top 3 → 예산 vs core_factor 대조:
  - 충족 → "케이스 X 유사 — 이 믹스·타임라인·앵글" 운영안 (WS7 브리프 재사용)
  - 미달 → "불가 — 코어가 [조건]" 또는 축소안 (must_have 최소선 기준)
- 산출: 운영안 md/HTML (WS7 파이프 재사용 — 신규 생성기 만들지 말 것).
- 완료 기준: 실제 진단 응답 1건(사용자 제공)으로 매칭 + 운영안 초안 1건. **판단 로직(가능/불가 경계)은 초안 후 사용자 검수 필수** — 이게 상품이 되는 지점이라 사람 사인 없이 배포 금지.
