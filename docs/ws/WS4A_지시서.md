# WS4a 지시서 — 화면 목업 우선 (사용자 검수용)

> 실행 세션용. **목적: 코드/데이터보다 화면 기획을 먼저 확정한다.** spec/03 정의서대로 화면을 목업 데이터로 렌더해서 사용자가 "직관적인가"를 실제 화면으로 검수할 수 있게 만드는 것이 전부다. 데이터 결선은 검수 확정 후 WS4b(기존 WS4_지시서)가 담당.
> 시작하면 `docs/spec/03_화면기능정의서.md` 전체(특히 §0 U1~U8·§0.1 적재 위저드)와 `docs/BP_재설계_v2.md` §1.2 Q1~Q7을 먼저 읽을 것.

## 공통 규칙
- 브랜치 `ws-4a-screens`에서 작업, 논리 단위 로컬 커밋. **push·배포·Supabase 마이그레이션·프로덕션 DB 쓰기 전부 금지.**
- **라이브 데이터 결선 금지** — 모든 화면은 정적 fixture로 렌더. 단 fixture는 지어내지 말고 **실제 ready 케이스 1개의 데이터를 read-only로 추출**해 JSON으로 저장해 쓸 것 (현실적인 분포·긴 텍스트·빈 값이 검수 품질을 좌우한다). 추천: 닥터리쥬올 케이스 또는 contents 수 상위 케이스.
- 기존 mockup 컴포넌트(MOCKUP_VS_LIVE_REPORT.md 참고)는 재사용 가능하되, U1~U8 위반이면 고친다.
- 완료 시 `docs/ws/WS4A_REPORT.md` 보고서 + 설계 문서 §6 로그 한 줄.

## 만들 화면 (라우트는 `/mockup/*` 밑에 격리 — 기존 화면 건드리지 않음)

### 1. 케이스 상세 (`/mockup/case`) — 메인
- **질문형 TOC** (U4): "언제 터뜨렸나 / 누구를 썼나 / 뭘 찍었나 / 팔렸나 / 광고는 어떻게 / 뭘 베낄까" — 알파벳 A~E/G 폐기.
- 섹션마다 **상단 1줄 결론** (U2) — fixture 기반 실제 문장으로 (예: "광고의 70%가 Amazon행 = Amazon 퍼널 중심"). 결론 못 만드는 블록은 회색 "데이터 없음".
- **프리셋 우선** (U1): 섹션별 디폴트 조합 1개만 렌더, 토글은 "자세히" 뒤로.
- 블록 구성은 spec/03 §2.3 표 그대로 (A: 월별+TT샵 분리·BSR 오버레이·변곡점 한 문장(U8) / B: 티어 분포·반복 협업·시딩∩광고 / C: 클러스터·티어×앵글×월·매출 기여 특징 / D: SKU·BSR 일원화 / E: 광고 랭킹·source_channel / G: 5축+core_factor 입력 UI).
- **영상 인라인 임베드** (U7): 클러스터 예시·top 작성자·변곡점 전후 최소 3곳에 실제 TikTok iframe 임베드로 시연.
- **완결성 게이지** 헤더: 6축(규모/구성/콘텐츠/성과/광고접합/시점) 충족 표시.
- 추정치 `~` 표기 + 신뢰도 배지 (U5), "광고 집행(스파크애즈)" 라벨 (U6), 내부 용어 0 (U3).

### 2. 적재 위저드 (`/mockup/intake`) — spec/03 §0.1
- 케이스 생성(브랜드+국가) 직후 화면: 자동 수집 시작 배너 + country×channel 기반 수동 재료 체크리스트(출처 링크·예상 소요·업로드 즉시 "n행 적재됨" 피드백) + "지금 올린 것으로 답할 수 있는 것: 6축 중 ②③" 게이지 연동.
- 업로드 인터랙션은 가짜(setTimeout)로 충분 — 흐름 검수가 목적.

### 3. 케이스 목록 (`/mockup/cases`)
- 완결성 게이지 요약 컬럼 + 필터(국가×판매채널×예산).

### 4. PhaseProgress (케이스 상세 내)
- 신 11 phase × {상태·비용·partial 잔여·재실행 버튼} — 사용자 언어로 ("영상 태깅 중 · 잔여 139건"), phase 코드 노출 금지.

## 완료 기준
- `npm run dev`로 4화면 전부 열림 + **화면별 스크린샷**(Playwright, 라이트 테마) REPORT 첨부.
- REPORT에 **검수 체크리스트**: U1~U8 각각 "어느 화면 어디서 확인 가능한지" + 스스로 판단한 위반 잔여 목록.
- REPORT에 **결정 요청 목록**: 정의서가 모호해서 임의로 정한 것들 (사용자가 목업 보면서 결정할 항목).
- 코드 diff만으로 완료 처리 금지 — 렌더된 화면 기준.
