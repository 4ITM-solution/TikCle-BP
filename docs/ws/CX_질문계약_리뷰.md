# CX-2 — Q1~Q7 질문 계약 비판 검토

범위: `docs/BP_재설계_v2.md` §1.0~1.4, `docs/spec/03`, `docs/spec/04`, 파일럿 D1. 기존 P/G, R규칙, QA 보고서와 중복되는 데이터 품질/구현 버그는 제외했다.

## 결론

Q1~Q7은 "한 케이스 안에서 무엇이 일어났는가"를 읽는 L1/L2 관측 질문으로는 대체로 타당하다. 그러나 §1.0의 제품 정의는 "필터를 통과한 온전한 케이스 몇 개를 제시하고, 진입 플레이북/운영안으로 연결"하는 참조 체계다. 현재 질문 계약은 케이스 내부 설명에 치우쳐 있고, 케이스를 고르고 버리고 권장안으로 바꾸는 L3 질문이 부족하다.

## 기준

- BP는 국가×판매채널×예산/스테이지 필터를 통과하는 온전한 케이스를 제시해야 한다: `docs/BP_재설계_v2.md:24-32`
- 완결성 6축은 케이스 설명력의 조건이다: `docs/BP_재설계_v2.md:36-47`
- 최상위 산출물은 진입 플레이북이다: `docs/BP_재설계_v2.md:56-68`
- Q1~Q7은 발행량, 반복 협업, 앵글, 성과 변화, GMV 특징, 광고 소재, 시딩∩광고를 묻는다: `docs/BP_재설계_v2.md:70-80`
- 진단-매칭 운영안은 필터→core_factor 판정→운영안 생성을 요구한다: `docs/spec/04_아웃풋_명세.md:41-62`

## 새로 발견한 계약 갭

### CX2-F1. "이 케이스를 채택/폐기해도 되는가?"를 묻는 질문이 없다

§1.0은 조각 단위가 아니라 케이스 단위 채택/폐기를 목표로 한다. 하지만 Q1~Q7은 모두 케이스가 이미 분석 대상이라는 전제에서 내부 현상을 묻는다. 완결성 게이지는 데이터 존재 판정이고, "레퍼런스로 채택 가능한가"라는 판단 질문은 아니다.

- 빠진 질문 제안: **Q0. 이 케이스는 어떤 필터 조합에서 레퍼런스로 채택/보류/폐기인가?**
- 필요한 입력: country/channel/예산티어/stage, 완결성 6축, core_factor 확정 여부, 데이터 최신성, 수동 원천 누락 여부.
- 산출: `adopt | hold | reject`와 사유. 브리프/진단 매칭 후보 진입 전 게이트로 쓰는 것이 맞다.

### CX2-F2. "비슷한 후보 중 어느 케이스를 보여줄 것인가?"가 없다

제품 정의는 필터 후 "온전한 케이스 몇 개"를 제시하는 참조 체계다. Q1~Q7은 단일 케이스 상세 질문이라 후보 랭킹, 대표성, 다양성, 중복 제거 기준을 제공하지 않는다.

- 빠진 질문 제안: **Q8. 같은 필터 조합에서 대표 케이스 top N은 무엇이고, 왜 이 순서인가?**
- 필요한 입력: 완결성 점수, 예산/스테이지 적합도, 성과 관측 가능성, core_factor 확정, 유사 케이스 간 중복도.
- 이유: 진단-매칭 운영안의 1단계가 "완결성 게이지 상위 케이스 후보"라고 되어 있으나, Q1~Q7만으로는 후보 순위의 설명 가능성이 약하다.

### CX2-F3. 예산/스테이지 정규화 질문이 Q1~Q7 내부에 없다

Q1은 기간별 발행량과 인원수를 묻지만, §1.0의 핵심 필터는 예산/스테이지다. 진입 플레이북은 "미국 시장에 이 규모로 진입하려면"이라고 표현되어 있으나, Q1~Q7에는 예산 대비 물량, 진입 N개월차, stage별 pace를 정규화하는 질문이 없다.

- 빠진 질문 제안: **Q9. 이 케이스의 월예산/진입 N개월차 기준 운영 강도는 얼마인가?**
- 필요한 입력: 수동 예산 proxy, 기간, 영상/인플/티어 믹스, 프로모션 캘린더, stage.
- 산출: `videos_per_budget_band`, `creators_per_budget_band`, `angle_sequence_by_month_since_entry`.
- 효과: Q1의 raw count를 플레이북의 "내 예산에서 가능한가" 판단으로 바꾼다.

### CX2-F4. "성공 조건의 최소 요건"을 검증하는 질문이 없다

spec/04는 `core_factor`를 사람 확정값으로 두고, 진단 응답과 대조해 가능/축소/불가를 판정한다고 한다. 하지만 Q1~Q7은 core_factor를 도출하거나 검증하는 질문이 아니다. 결과적으로 브리프는 설명을 할 수 있어도 운영안은 사람의 사후 해석에 크게 의존한다.

- 빠진 질문 제안: **Q10. 이 케이스가 작동한 최소 조건과 대체 불가능한 조건은 무엇인가?**
- 필요한 입력: Q1~Q7 결과, 예산/스테이지 정규화, 성과 변곡 전후 비교, 광고/시딩 접합 여부.
- 산출: `core_factor_candidate`, `must_have_conditions`, `nice_to_have_conditions`, `not_transferable_risks`.
- 비고: LLM 초안 후 사람 확정이라는 spec/04 방향은 유지하되, Q계약에 core_factor 검증 질문을 승격해야 한다.

### CX2-F5. 리프레시/최신성 질문이 Q계약 밖에 있다

파일럿 D3에는 재수집 주기와 델타 크기 실측이 있다. 그러나 Q1~Q7 계약에는 "이 답은 언제까지 유효한가"가 없다. 광고 생존편향(G3)처럼 라벨링 이슈는 등록되어 있지만, 질문 계약 수준에서는 freshness가 답의 일부가 아니다.

- 빠진 질문 제안: **Q11. 이 케이스 답변의 최신성/관측 윈도우/다음 리프레시 필요일은 언제인가?**
- 필요한 입력: source별 last_seen/collected_at/imported_at, delta size, active observation period.
- 산출: source별 freshness badge와 "지금 답해도 되는 질문/보류할 질문" 목록.
- 이유: §1.0.3은 자동 수집분으로 당일 1차 브리프를 허용한다. 그러려면 답마다 최신성 한계가 계약에 들어가야 한다.

## 불필요하거나 합쳐도 되는 질문

### CX2-N1. Q5는 독립 질문이라기보다 Q3×Q4의 절단면이다

Q5 "매출 기여 추정 콘텐츠의 특징"은 Q3의 콘텐츠 특징과 Q4의 성과 변곡/GMV를 조인한 결과다. 독립 질문으로 두면 `영상별 GMV`가 없는 케이스에서 항상 구멍이 나며, "매출 기여"라는 강한 표현 때문에 추정 중첩(G4) 리스크도 커진다.

- 제안: Q5를 독립 상위 질문에서 내리고 Q3/Q4의 하위 렌즈로 둔다.
- 문구: "성과와 함께 움직인 콘텐츠 특징"처럼 인과보다 관측 상관에 가까운 표현이 안전하다.

### CX2-N2. Q6과 Q7은 "광고 접합" 하나의 상위 질문으로 묶는 편이 낫다

완결성 6축의 ⑤는 광고 접합이다. Q6은 광고 소재, Q7은 시딩∩광고 인플이라 둘 다 같은 축의 부분 질문이다. 별도 질문으로 두면 E 섹션이 과대 대표되고, Q0/Q8/Q9 같은 선택/정규화 질문이 밀린다.

- 제안: 현재 Q6/Q7을 **Q6. 광고는 시딩 자산을 어떻게 재활용했나?**로 통합한다.
- 하위 렌즈: 최장 운영 소재, origin/source_channel/banner_style, 겹치는 인플과 앵글.

## 제안 질문 계약 v2

| # | 질문 | 역할 |
|---|---|---|
| Q0 | 이 케이스는 어느 필터에서 채택/보류/폐기인가? | 케이스 후보 게이트 |
| Q1 | 기간×채널×진입 N개월차 기준으로 얼마나 운영했나? | 규모/시점 |
| Q2 | 누구를 반복적으로 썼고, 티어/채널 구성은 무엇인가? | 구성 |
| Q3 | 어떤 앵글/USP를 어떤 순서로 썼나? | 콘텐츠 |
| Q4 | 발행/광고 변화와 매출·BSR·GMV가 언제 함께 움직였나? | 성과 |
| Q5 | 광고는 시딩 자산을 어떻게 재활용했나? | 광고 접합 |
| Q6 | 이 케이스의 최소 성공 조건과 이전 불가능한 조건은 무엇인가? | core_factor |
| Q7 | 같은 필터 조합의 대표 케이스 top N은 무엇인가? | L3 매칭 |
| Q8 | 이 답은 얼마나 최신이며 어떤 축은 보류해야 하나? | freshness |

이렇게 바꾸면 기존 Q1~Q7의 대부분 데이터 투자는 유지하면서, §1.0의 케이스 참조 체계와 spec/04의 진단-매칭 운영안까지 질문 계약이 직접 연결된다.
