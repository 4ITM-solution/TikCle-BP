# BP 정보 니즈 지도 v2 — 기본 단위 × 조합 × 화면/질문 경계

> 핵심 진단: **현행 대시보드는 "축 1개 = 섹션 1개"(A=시간·B=크리에이터·C=콘텐츠·D=SKU·E=광고)인데,
> 사용자의 실제 질문은 예외 없이 "축 2~3개의 교차 + 지표 병치"다.** 섹션 경계를 넘는 조합이
> 필요한 순간 화면이 답을 못 하고 전부 분석 세션으로 흘러간다. 프라임 리포트와 내러티브
> 탐색기는 별개 기능 요청이 아니라 이 구조 갭의 증상 2개다.

## 3. 경계 — 화면 vs 질문

```
그 질문의 답이…
├─ DB 사실의 재조합            → 화면 (조합 뷰)          : C4·C5·C7후보·C8
├─ 입력(등록/적재)만 있으면 자동  → 화면 + 빈상태 유도       : 이벤트 등록·매출 주기적재·예산 proxy
└─ 확정·해석·외부검증 필요      → 애널리스트 (매번 질문)    : 판정 서술·C7 확정·C12·수집 밖 실사·데이터 의심 검증
```
승격 루프: 애널리스트가 반복해 푼 방법은 정형화되어 화면으로 내려온다 (계보 깔때기·키 메시지).
INSIGHT_INBOX → 배차 루프가 그 통로 — 유지.
