📋 개선 체인지로그 13건 펼치기 (블록별 · 배포판→개선)
  1. 헤더 완결성 게이지 행 + 최신성 행 2줄 → 신뢰 정보 1줄 통합 (게이지·6축·freshness·캐시 배지 전부 유지)
  2. Phase 패널 배포판은 PhaseProgress + PhaseRunsPanel 2중 → 1패널 통합 (phase별 상태·수치·재실행 ↻ 유지) — "한 사건은 화면에 한 번만"
  3. 데이터 채널 상시 펼침 7카드 그리드 → 기본 접힘 콘솔 (요약줄 상시, 클릭 1번에 전체 — 상단 스트립 dot과 중복이라 분석 흐름에서 분리)
  4. A 차트 오버레이 라인이 상단 밴드[28,158]에 갇혀 실제 값과 무관한 높이로 그려짐 → 실측 스케일 + 축 라벨 (영상수=막대와 동일 축, BSR·광고%=좌/우 축 눈금)
  5. A KPI gifted(시딩) 상시 "-" → C의 seeded regex 집계 연결 표시 (추정 라벨 부착)
  6. B 채널 토글마다 표 소스가 조용히 바뀜 → 표 위에 현재 스코프 문장 캡션 ("지금 보는 풀: …")
  7. B Top 작성자 기본 5명 → 기본 10명 (더보기 20명 유지 — 정보 증가)
  8. C 탭 7개 이름만으로 구분 어려움 → 활성 탭 아래 "이 탭이 답하는 질문" 한 줄 부착 (탭 수·내용 유지)
  9. D "전체 브랜드" 지표와 "채널·SKU" 지표가 한 흐름에 섞여 스코프 혼동 → ① 브랜드 전체 / ② 채널·SKU 상세 2밴드 분리 (내용 전량 유지)
  10. D Affiliate code conversion 상시 빈 탭 → E의 promo code 추출 재노출 + E 점프 링크
  11. E KPI 6개가 5열 grid라 마지막 카드 홀로 줄바꿈 → 6열 정렬
  12. 공통 섹션 결론 한 줄·드릴다운↗·임베드·표본/추정 라벨 — 배포판 문법 그대로 유지 (변경 없음)
  13. ★신규: 전역 분석 기간 필터 — 헤더에서 기간 선택 시 A·C·E·KPI가 즉시 재집계 (유료 재실행 $0). 티어·태그·클러스터 소속은 영상/인플 단위 속성이라 그대로 필터만 됨. 클러스터 정의문·USP 사전 등 "전 기간 코퍼스 산출물"은 라벨로 구분. 매출(D)은 30d 스냅샷/Kalodata 기간 적재 별도 축
