# BP 6개월 로드맵 (2026-07 ~ 2026-12)

> 2026-07-07 작성. 기준 문서: `BP_재설계_v2.md`(제품 정의·WS 정의), `spec/`(현상태 명세).
> 목표 한 줄: **"필터를 통과하면 온전한 케이스가 나오고, 그 케이스가 브리프·진단·플레이북으로 팔리는 상태"** (§1.0)를 6개월 안에 만든다.

## Phase 2 — 케이스 공장 가동 (8월) ✦ 커버리지 플랜 §1.0.4 시작

시스템 개발 → 케이스 생산으로 무게 이동. **이 과정 자체가 파이프라인 실전 E2E다.**

- **미국 아마존 6케이스** 완결성 6축 충족 (닥터리쥬올 포함, 예산 규모별 + 프라임데이 시점별)
  - 케이스당: 자동 수집 즉시 + 수동 재료(Exolyt·Helium·Keepa) 위저드 턴어라운드 1~2일
  - 비용: ~$6/케이스 (티어링 후) — 6케이스 ~$40
- **WS9 딥 비전 & 매칭 v2** (8월 중순~): 컷 단위 딥 태깅 L2(~$6/케이스)·SKU 신호계단+커버리지 표기·광고↔원본 워터마크 매칭
  - "비전이 얕다 / SKU 귀속 안 된다 / 광고 원본 못 찾는다" — 사용자 3대 문제 제기의 해소 지점
  - 게이트: 실케이스 1개 딥 100영상 사람 채점 통과
- 운영 리듬 확립: 주간 완결성 게이지 리뷰 → 부족 축 데이터 요청 → 증분 phase

## Phase 3 — L3 합성 (9월) ✦ WS6, 첫 "진입 플레이북"

케이스 2개+ 모이면 횡단이 가능해진다.

- compare 페이지 정규화 축: **진입 후 N개월차·예산 티어·티어 믹스·앵글 시퀀스**
- 케이스 필터 (국가×판매채널×예산) + 완결성 게이지 기반
- **cases.core_factor**: 케이스 작동 핵심조건 한 줄 — LLM 초안 → 사람 확정 UI (WS8의 매칭 근거)
- 완료 기준: 케이스 2개+로 플레이북 초안 1개 + 지역 단위 질문("미국 아마존 진입 핵심?") 답변 1건

## Phase 4 — 브리프 상품화 (10월) ✦ WS7, 가공 A

대시보드(원료) → 꺼내 쓰는 물건으로.

- 케이스 브리프 생성기 3종: 슬랙 요약(5줄) / 세일즈 Appendix(케이스 스토리) / 미팅 브리프(질문 대응형)
- 원칙: **수치는 뷰에서 템플릿 주입, LLM은 서술만** (수치 지어내기 원천 차단)
- 톤 레퍼런스: 월간 사보 html·닥터포헤어 미팅 html·4am.team/blog
- GTM 콘텐츠 캘린더 루프와 접합 — 케이스 브리프가 주간 콘텐츠 소스로 흐르게
- 완료 기준: 실케이스 1개로 3종 생성, 수치 전부 뷰 라이브 검증

## Phase 5 — 진단-매칭 퍼널 (11월) ✦ WS8, 가공 B

진단 → 구독 퍼널의 상품 엔드포인트. **선행: WS6(core_factor)·WS7(브리프).**

- tikcle-result 응답(국가·판매채널·예산·스테이지) → 케이스 필터 매칭
- 판단 로직: "케이스 X와 유사" + 예산이 core_factor 충족 → 운영안 / 미달 → "불가 — 코어가 [조건]이라서" 또는 축소안
- 완료 기준: 실제 진단 응답 1건으로 매칭 케이스 + 운영안 초안 생성

## Phase 6 — 확장·운영 자동화 (12월)

- 커버리지 확장: 미국 틱톡샵 ~3개 → 영국 아마존 (동남아는 데이터 소스 가용성 재확인 후)
- **G3(광고 생존편향) 데이터 해소**: /monitoring 주기 스냅샷이 6개월치 쌓이는 시점 — 운영일수 랭킹이 "관측 기반"으로 승격
- 운영 자동화: BSR 정기 갱신·케이스 신선도 리프레시·월간 비용 리포트(가드 데이터 활용)
- 회고: v1 대비 비용·안정성·턴어라운드 총결산
