## Phase 2 — 케이스 함대 전체 리프레시 + 공장 가동 (7월말~8월) ✦ 개정 2026-07-07

> 사용자 정정: "신규 6개"가 아니라 **기존 93케이스(ready 82) 전체가 새 파이프라인 기준으로 업데이트**되는 게 우선.

**2-A. 전 함대 일괄 리프레시 (자동 축 — 크레딧만 있으면 즉시, 웨이브 배치)**

- collect-meta 재수집(upsert 안전, 최신 광고 반영) + enrich 공백 채우기(조건부 enrich 조건 하)
- 완결성 게이지 스냅샷 → **부족 축 매트릭스** (어느 케이스가 어느 축이 비었나 한 장)
**2-B. 수동 축 보충 (사람 병목 — 게이지 우선순위순)**
- Helium·Keepa·Exolyt 최신 export는 계정 접근이 필요해 일괄 자동화 불가 — 게이지 매트릭스가 "이번 주 뽑을 것" 목록이 됨
**2-C. 신규 커버리지** (§1.0.4): 미국 아마존 예산 규모별 + 프라임데이 시점 신규 케이스 — 리프레시된 함대 위에 추가
**2-D. WS9 딥 비전 & 매칭 v2**: 컷 단위 딥 태깅 L2(~$6/케이스)·SKU 신호계단·광고↔원본 매칭. 게이트: 실케이스 1개 사람 채점

## Phase 4 — GTM 발행 레이어 (9월말~10월) ✦ WS7 확장판, 가공 A (개정 2026-07-07)

> 사용자 정정: 브리프 3종을 넘어 **이미 코워크/코덱스에서 분절 운영 중인 GTM 발행 워크플로우들을 BP에 접합**한다. 실증 사례 = 스페인어권 플레이북 v7 (슬랙 방향 → 웹리서치 + **BP DB 실측** → 기획 md 2종 → pptxgenjs 덱 → 사람 폴리시).

**공통 파이프 구조** — 모든 발행물이 같은 4단계를 탄다:
```
① 소스     BP 뷰 실측(수치 자동 주입) + 웹리서치 + 슬랙 방향성
② 기획 md  섹션↔데이터 매핑 + 부족정보 리스트  ← 사람 확정 체크포인트 1
③ 생성     코드(pptxgenjs·HTML 템플릿·md) — LLM은 서술만, 수치는 뷰에서
④ 폴리시   톤·카피 최종                        ← 사람 확정 체크포인트 2
```

**발행물 3종 (실존 워크플로우의 시스템화):**
| 발행물 | 현재 상태 | 시스템화 내용 |
|---|---|---|
| **TikCle 레터** (월간 뉴스레터) | tikcle-letter 스킬 + SES 셀프호스팅 발송·추적 인프라 가동 중 | 딥다이브 코너 = BP 케이스 브리프 자동 공급. 스킬은 `docs/gtm-assets/tikcle-letter/`로 이관 완료(유실 방지) |
| **지역/언어권 플레이북 덱** | 스페인어권 v7 (수동 5단계) | "언어권/지역 컷" 표준 쿼리 세트(콘텐츠 수·티어 풀·나노 vs 메가 성과·케이스 목록) + 기획 md 템플릿 + build_deck 재사용. 레퍼런스 `docs/gtm-assets/스페인어권_플레이북_인덱스·덱기획.md` |
| **케이스 보고서 HTML** | 와이어트용(오가나셀·닥터리쥬올) 등 수동 제작 이력 | WS7 브리프 생성기의 "미팅 브리프" 확장 — 케이스 뷰 → HTML 보고서 |
| (기존) 브리프 3종 | — | 슬랙 요약 / 세일즈 Appendix / 미팅 브리프 — 위 3종의 최소 단위 |

- 원칙: **수치는 뷰에서 템플릿 주입, LLM은 서술만** (수치 지어내기 원천 차단)
- 자동화하지 않는 것(설계에 명시): 방향 결정(슬랙)·기획 md 확정·최종 폴리시 — 사람 체크포인트 2개는 워크플로우의 일부다
- 완료 기준: 실케이스 1개 → 레터 딥다이브 1편 + 지역 컷 쿼리로 덱 기획 md 1개 + 보고서 HTML 1개, 수치 전부 뷰 라이브 검증
