# TikCle 레터 (새벽네시 월간 뉴스레터)

매달 새로 포맷을 짜는 건 낭비다. 이 스킬은 확정된 구조·톤·발송용 HTML 템플릿을 재사용해서, 그달 재료만 넣으면 레터가 나오게 한다.

## 이 레터가 뭔지 (북극성)

이은솔 대표 컨셉: **"새벽네시가 이번 달 본 것."** 대단한 새 소스가 아니어도, 그달 시장에서 관찰한 걸 묶어서 가볍게 보낸다. 목적은 **영업 너처링** — 대놓고 파는 게 아니라, 부담 없이 읽히고 필요할 때 우리를 떠올리게. 계속 보내다 보면 그중 하나가 맞는다.

- 성격: 가벼운 큐레이션 + 한 꼭지 딥다이브(벤치마크). 3~4분 분량.
- 주기/채널: 월 1회, 이메일(스티비 등) + 사내 슬랙 공유.
- OSMU 종착점: 그달 블로그·쓰레드·웨비나를 '묶어서' 재가공 → 새 글 안 써도 레터가 나온다.
- 발신 주체: **TikCle 팀**("안녕하세요, TikCle 팀이에요 :)"). 회사 주체를 말할 땐 '새벽네시', 서비스는 'TikCle'. 섞지 말 것.

## 작업 순서

1. **재료 모으기 (혼합 입력).** 아래 "입력" 참고. 딥다이브 브랜드/케이스는 사용자에게 받고, 시장 시그널은 커넥터가 있으면 수집 시도하고 없으면 사용자에게 묻는다.
2. **구조 잡기.** `references/format.md`를 읽고 그달 재료를 고정 코너에 배치. 실제 발송본 흐름은 `references/example-6wol.md`(원고)와 `references/example-6wol-email.html`(HTML)이 골드 스탠다드다. 새 레터를 쓰기 전에 이 예시를 반드시 먼저 열어 톤·길이·끊는 지점을 참고할 것.
3. **원고(md) 작성.** format 구조 + 아래 톤 규칙으로.
4. **발송용 HTML 생성.** `assets/build_email_html.py`에 콘텐츠 dict를 채워 실행 → HTML 파일 산출. 스크립트 상단 docstring에 dict 스키마와 예시가 있다.
5. **발송 전 체크리스트**(맨 아래) 확인하고 사용자에게 원고 + HTML 둘 다 전달.

## 입력 (혼합)

- **사용자에게 받는 것 (핵심)**: 이 달의 딥다이브(디깅) 브랜드/케이스와 그 근거 데이터. 이게 레터의 메인이라 실측/실명 근거가 있어야 신뢰가 산다. 없으면 먼저 요청할 것.
- **스킬이 수집 시도하는 것 (있으면)**: 그달 시장 시그널 3개. 소스 = 메일함 뉴스레터(트렌디어·뷰티누리·CGETC 등), 슬랙 `#4am_gtm-contents`·`#tikcle_gtm-sales`, 4am.team 블로그·쓰레드. Gmail/Slack 커넥터가 연결돼 있으면 최근 한 달 신호를 훑어 초안 3개를 뽑고, 없으면 사용자에게 "이번 달 주목한 시장 신호 있어?"라고 묻는다. 각 시그널은 사실 2~3줄 + **"왜 중요한가" 한 줄**.
- **그달 자사 콘텐츠**: 딥다이브 블로그 링크, 웨비나 신청 폼/일정, 신규 블로그. 재사용(OSMU)이 핵심이니 그달 블로그·쓰레드·웨비나를 먼저 확인.

## 고정 구조 (실제 발송본 기준)

순서대로. 세부 카피 규칙은 `references/format.md`.

1. **다크 헤더** — "TikCle Letter" / 발행 시점(예: 2026년 6월 3주차) / 한 줄 소개.
2. **인트로** — "안녕하세요, TikCle 팀이에요 :)" + 이번 호 한 줄 관점(이번 달 저희가 주목한 건 OO).
3. **이번 달 새로운 시장 소식** — 그달 가장 큰 시장 변화 1개를 조금 깊게(+ 나라별 차이 같은 디테일). 여러 개면 시그널 3 리스트로.
4. **이번 달 디깅 브랜드 (벤치마크·메인)** — 라일락 박스 안에 ①②③ 구조로 브랜드가 실제로 어떻게 하는지 해부 → `❌ 통념 → ✅ 진짜` 정리 → "👉 우리 브랜드 점검" 질문. 실명·실측·크리에이터 핸들/팔로워(스냅샷 기준).
5. **웨비나 배너 (선택)** — 그달 웨비나 있으면 다크 배너 + 신청 버튼. 없으면 생략.
6. **더 깊게 읽어보기** — 그달 딥다이브 블로그 1편 요약 + "블로그에서 읽기 →" 버튼.
7. **새소식** — 웨비나·신규 블로그·서비스 업데이트 번호 다이제스트 1~3개.
8. **마치며** — 이번 호 한 줄 결론 + 가벼운 CTA(TikCle 진단서 14문항 / 커피챗). 하드셀 금지.
9. **푸터** — 사업자 정보(주식회사 새벽네시 · 대표 이은솔, 김경은 · 사업자등록번호 111-81-94199 · 서울 강남구 논현동 158-13, 3-4F) + 문의 team-og@4am.team + 수신거부.

## 톤

- 블로그 해요체보다 **더 가볍고 사람 냄새.** "이번 달 저희가 본 건…", "이거 재밌었어요", "신기하죠 ㅎㅎ". 큐레이터/동료 시점.
- 자랑·영업 톤 최소화. 가르치지 말 것.
- 이모지 가볍게(📍🔍🍓👀), 과하지 않게.
- 벤치마크 정리는 `❌ 통념 → ✅ 진짜` 대구로 각인시키기.

## 반드시 지킬 것 (컴플라이언스·포맷)

- **앰대시(—) 금지.** 가운뎃점(·)·쉼표로.
- **표 금지 → 전부 리스트.** 이메일/웹플로우 렌더 호환 때문(HTML 빌더도 표 대신 `•`/줄바꿈 리스트 사용).
- **명칭**: 서비스=TikCle, 회사=새벽네시. 섞지 말 것.
- **아마존 표현**: "공식 파트너/협력사" 금지 → "아마존 외부 서비스 사업자(Service Provider)". Amazon 로고 단독 사용 금지(Amazon Global Selling 로고만).
- **실명·수치**: 벤치마크 실명 표기는 사용자 확인 후. 크리에이터 팔로워는 "스냅샷 기준"임을 명시.
- **링크 플레이스홀더**를 남기지 말 것 — 딥다이브 블로그 URL, 웨비나 신청 폼, 진단서/문의 링크를 사용자에게 받아 채우거나, 없으면 눈에 띄게 `[링크 필요]`로 표시하고 발송 전 체크리스트에 올린다.

## 산출물

1. `레터_YYYY년_M월호.md` — 마크다운 원고(사내 슬랙 공유·검수용).
2. `레터_YYYY년_M월호_email.html` — 발송용 반응형 HTML(`assets/build_email_html.py`로 생성).

## 발송 전 체크 (내부)

- 링크 다 채웠나: 딥다이브 블로그, 웨비나 신청 폼, 진단서/문의.
- 실명 표기 OK 확인. 크리에이터 팔로워 스냅샷 기준 명시.
- 표 없이 전부 리스트인지(이메일 호환).
- 앰대시 없는지, 명칭(새벽네시/TikCle) 안 섞였는지, 아마존 표현 규칙 지켰는지.
- 길이: 시그널은 더 줄여도 됨. 벤치마크가 그달 메인.
- 발송은 사람이(스티비/Gmail). 스킬은 원고+HTML까지만.
