## 🐛 실제 코드 버그 (수정 완료)

### 1. Phase 4c.5 IG 팔로워 enrich — 타임아웃 → Apify 중복 과금 루프
- **증상**: IG author 많은 케이스에서 분석 죽거나 Apify 비용 중복.
- **원인**: `enrichIgAuthorFollowers`가 비-durable `runIgProfileScraper`(폴링 20분)를 `step.run` 안에서 호출 → Vercel maxDuration(800s) 초과 → 함수 강제종료 → Inngest 재시도 → 새 Apify run(중복 과금) 반복. 같은 phase4c의 다른 스크래퍼는 durable인데 이것만 누락.
- **수정**: `runIgProfileScraper`/`enrichIgAuthorFollowers`에 durable 옵션 추가, run-analysis에서 step.run 언랩 + step 전달. (3파일)

### 2. Meta 광고 결합 파이프라인 부재 (엔진 업그레이드)
- **증상**: 단일 스크래퍼라 데이터 한쪽 손실 — 공식액터는 파트너십 attribution은 주지만 종료광고 미흡, curious_coder는 종료광고(효율)는 주지만 파트너십 없음. + 키워드 스크랩엔 리테일러·카피 섞임.
- **수정**: `fetchMetaAdsCombined` — 키워드(종료+활성) + 공식액터(파트너십) merge, ad_archive_id로 합치고 리테일러·카피 자동 제외. phase4a·모니터링 트래커가 이걸 씀. `is_brand_official`을 page_id 기준으로 정확 판정.
- **검증**: Mixsoon — page_id 자동도출(mixsoon US) ✓, 종료132+활성32 ✓, 잡것 제외 ✓.
